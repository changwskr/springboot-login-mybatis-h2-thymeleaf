Spring Boot Login Application 홈페이지

🏠 HOME | 사용자목록(1번 DB) | 사용자목록(2번 DB) | 통합사용자관리 | 예금관리 | 거래이력 | 토큰생성 | SWAGGER | ACTUATOR LOGOUT

원하는 사용자 목록, 예금 관리, 거래 이력, 토큰 생성, Swagger 또는 Actuator를 선택하세요.

사용자 관리:

  • 사용자 목록 (1번 DB) - DB1 사용자 조회 및 관리
  • 사용자 등록 (1번 DB) - DB1에 새 사용자 등록
  • 사용자 목록 (2번 DB) - DB2 사용자 조회 및 관리
  • 사용자 등록 (2번 DB) - DB2에 새 사용자 등록
  • 통합 사용자 관리 - 두 DB의 사용자 통합 관리
  • 통합 사용자 등록 - 두 DB에 동시 등록

예금 관리:

  • 예금 목록 - 모든 예금 조회
  • 예금 등록 - 새로운 예금 등록

거래 이력 관리:

  • 전체 거래 이력 - 모든 거래 내역 조회
  • 터미널별 통계 - 터미널별 거래 수 통계
  • 요청명별 통계 - 요청 유형별 통계

관리자 기능:

  • 토큰 생성 - 관리자 토큰 생성
  • Swagger UI - REST API 문서

시스템 정보:

  • Health Check - 시스템 상태 확인
  • REST Health Check - REST API 상태 확인

REST API 엔드포인트:

사용자 관리 API:

  • DB1 사용자: /api/users/db1
  • DB2 사용자: /api/users/db2
  • 통합 사용자: /api/users/sync
  • 기본 사용자: /api/users

예금 관리 API:

  • 예금 목록: GET /deposite/list
  • 예금 등록 폼: GET /deposite/form
  • 예금 등록: POST /deposite/save
  • 예금 상세보기: GET /deposite/view/{id}
  • 예금 삭제: POST /deposite/delete/{id}

거래 이력 API:

  • 전체 거래 이력: GET /api/common/all
  • 사용자별 이력: GET /api/common/user/{userID}
  • 터미널별 이력: GET /api/common/terminal/{terminalID}
  • 거래번호 조회: GET /api/common/transaction/{transactionNo}
  • 요청명별 조회: GET /api/common/request/{reqName}
  • 터미널별 통계: GET /api/common/stats/terminal
  • 요청명별 통계: GET /api/common/stats/request

인증 API:

  • 로그인: POST /api/auth/login
  • 로그아웃: POST /api/auth/logout
  • 상태 확인: GET /api/auth/status

관리자 API:

  • 토큰 생성: POST /api/admin/token/generate
  • 토큰 상태: GET /api/admin/token/status

시스템 API:

  • Health Check: GET /api/health

참고: 모든 REST API는 JSON 형식으로 요청/응답하며, Swagger UI에서 상세한 API 문서를 확인할 수 있습니다.
